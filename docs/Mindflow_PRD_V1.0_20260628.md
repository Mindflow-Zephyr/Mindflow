## 1. 产品概述

### 1.1 产品定位

MindFlow 是一款个人生活与效率管理 App，整合 **待办**、**领域任务**、**生活领域**（穿搭、饮食、居住等）三大能力，帮助用户在单一应用中管理「今天要做什么」「项目级任务」以及「长期生活兴趣与资产」。

### 1.2 核心价值

- **待办**：日程、循环习惯、时间投入追踪
- **领域**：树形项目/子任务管理
- **生活**：分类化的兴趣与生活资产管理（穿搭衣橱、烹饪菜单、游戏/探店等）
- **本地持久化**：核心数据通过 SwiftData 保存在设备本地，日常使用无需联网

### 1.3 设计基调

- 绿色系渐变背景（白 → `#d8f3dc`）
- 玻璃态底部导航栏，选中 Tab 胶囊展开动画
- 卡片式布局，统一圆角与轻阴影
- 仅支持竖屏

## 3. 功能模块需求

### 3.1 待办模块

#### 3.1.1 模块目标

管理单次与循环待办，追踪时间投入与完成状态，提供今日/本周维度的效率反馈。

#### 3.1.2 列表与交互

- **今日待办 / 今日完成** 分区展示，支持切换
- 按时间段（`periodBucket`）分组展示
- 空状态轮播文案（如「享受属于你的时间」「生活需要一点留白」）
- **左滑删除**、**右滑完成/恢复**，带整卡滑出动画
- 点击进入待办详情页，可编辑备注、时间、重复规则等

#### 3.1.3 待办数据字段

| 字段类别 | 字段说明 |
|----------|----------|
| 基础 | 标题、备注、是否完成、工作状态（未开始/进行中/暂停/已完成） |
| 优先级 | P1–P4，支持循环切换 |
| 分类 | 工作/生活/健身/穿搭等（`TodoLifeCategoryCatalog` 映射） |
| 时间 | 创建时间、计划日期、结束日期、完成日期 |
| 时段 | 开始/结束时刻（时+分）、计划时刻 |
| 耗时 | 完成耗时（秒）、首次开始计时时刻 |
| 重复 | 重复模式（无/周/月/年/自定义） |
| 周重复 | 周几集合（1=周一 … 7=周日） |
| 月重复 | 每月第几天；29/30/31 是否回退到月末 |
| 年重复 | 月-日集合 |
| 自定义 | 间隔数值 + 单位（年/月/周/天/小时） |
| 循环限制 | 无限 / 截止日期 / 最大次数 |
| 循环状态 | 进行中 / 暂停 |
| 统计 | 累计完成次数、完成率分母 |

#### 3.1.4 计时能力

- 单条待办支持开始/暂停计时，光带动画反馈运行状态
- 完成时根据计时器与创建时间写入 `completionDurationSeconds`
- **穿搭类**待办完成时，耗时累计至「穿搭投入时间」（UserDefaults）

#### 3.1.5 汇总卡片

| 卡片 | 功能 |
|------|------|
| **今日投入** | 当日已完成待办总耗时；与昨日对比增量 |
| **下一项推荐** | 可切换推荐类型：最优先 / 最紧急 / 穿搭 / 运动 / 游戏；各类型独立主题色胶囊 |
| **本周趋势** | 7 日投入柱状图；底部「本周已完成」浅绿居中统计 |
| **循环待办** | 按分类分组；展示进行中/暂停/已完成汇总；可进入循环列表 |

#### 3.1.6 新建与编辑

- 底部面板：标题、计划日期时间、计划时长、分类、优先级
- 详情页：备注编辑、时间槽滚轮、重复规则配置（含自定义间隔滚轮）
- 新建第一条待办时，今日待办卡片有「小→大」生长动画

#### 3.1.7 数据持久化

- **SwiftData** `SDTodoRecord` 持久化
- **内存** `TodoActiveWorkState`：计时运行态，App 重启后重置

### 3.2 领域模块

#### 3.2.1 模块目标

管理有层级关系的项目任务，支持父任务与子任务树形展示。

#### 3.2.2 功能清单

| 功能 | 说明 |
|------|------|
| 任务列表 | 展示根任务，递归构建子任务树 |
| 新建任务 | 根任务或指定父级下的子任务 |
| 完成切换 | 切换任务完成状态；完成父任务时级联完成所有子任务 |
| 删除 | 删除任务及其整棵子树 |
| 子任务入口 | 从父任务进入「新建子任务」 |

#### 3.2.3 任务数据字段

- `id`（Int）、`title`、`description`、`isCompleted`、`parentId`
- 数据库存扁平结构；UI 层构建树形 `subtasks`

#### 3.2.4 数据持久化

- **SwiftData** `SDTaskRecord`
- `nextTaskId` 由 `SDAppMetadataRecord` 维护

### 3.3 生活模块

#### 3.3.1 模块目标

按多级分类组织兴趣与生活资产，对「穿搭」「烹饪」等场景提供垂直能力。

#### 3.3.2 分类树

| 能力 | 说明 |
|------|------|
| 分类 CRUD | 标题、图标、主题色、父子关系 |
| 导航 | 面包屑；进入子分类 |
| 删除 | 删除分类树（含子分类与关联条目）；支持滑出动画 |
| 样本结构 | 穿搭；饮食（烹饪/探店/游戏→在线游戏/单机游戏）；居住；旅行 |

#### 3.3.3 通用详情条目

适用于非穿搭/非烹饪的叶子分类：

- 字段：`title`、`note`
- 支持增删改、滑出删除

#### 3.3.4 穿搭子模块（分类名 =「穿搭」）

**衣物库**

- 筛选：大类 / 小类两行 Chip（上装/下装/鞋子/配饰及子类型）
- 排序：价格 / 穿着次数 / 颜色 / 品牌
- 衣物字段：名称、品牌、颜色、面料、季节、购买价、购买日期、穿着次数、最后穿着日
- 喜爱度五维评分：颜值、面料舒适度、合身度、质感、个人喜爱度 → 综合分

**排行榜**

- 喜爱度 / 价格 / 穿着次数 / 每次穿着费用 / 品牌持有数 / 连续穿着天数
- 品牌榜独立展示逻辑

**OOTD**

- 按槽位组合当日方案：上装 / 下装 / 鞋子 / 帽子 / 配饰
- 保存至日历历史；历史日期可查看、删除
- 穿着次数与 `lastWearDate` 回写逻辑（含待生效记录）

**页面设置**

- 可开关：衣物库 / OOTD 计划 / OOTD 日历 / 喜爱排行 / 操作卡片

**穿搭投入时间**

- 展示累计时长（UserDefaults）
- 与待办页「穿搭类」完成耗时联动累加

#### 3.3.5 烹饪子模块（分类名 =「烹饪」）

**菜单库统计卡（独立卡片）**

- 掌握总数、周增量、招牌/熟练/练习/想学习各状态数量

**菜单库**

- 字段：菜名、菜系、熟练度状态、制作次数
- 筛选：大类/小类两行 Chip（中餐/西餐/日韩/烘焙/其他）
- 状态：**招牌菜** / 熟练菜 / 练习中 / 想学习

#### 3.3.6 数据持久化

| 实体 | 说明 |
|------|------|
| `SDLifeCategoryRecord` | 生活分类 |
| `SDLifeDetailItemRecord` | 通用详情条目 |
| `SDWardrobeItemRecord` | 衣物（含喜爱度分数） |
| `SDMenuItemRecord` | 菜单项 |
| `SDOutfitPlanRecord` | 当前 OOTD 方案（JSON） |
| `SDOOTDHistoryRecord` | OOTD 历史 |
| `SDOutfitPageSettingsRecord` | 穿搭页卡片开关 |

## 4. 数据架构

### 4.1 持久化策略

| 数据 | 存储 | 备注 |
|------|------|------|
| 生活分类/衣物/菜单/OOTD | SwiftData | 首次启动种子数据，`hasSeeded` 标记 |
| 待办 | SwiftData | 增删改后自动 save |
| 领域任务 | SwiftData | 扁平存储 |
| 穿搭投入时间 | UserDefaults | 键 `outfitResearchTimeSeconds` |
| 待办计时状态 | 内存 | 重启丢失 |
| API 商品体系 | Mock | `CategoryDetailView` + `APIClient`，未接入主流程 |

### 4.2 数据生命周期

- **重新编译 / Run**：不删除 SwiftData 数据
- **删除 App / 重置模拟器**：数据清空
- **首次安装**：写入样本数据（分类、衣物、菜单、待办、任务）

### 4.3 关键 SwiftData 实体

- `SDAppMetadataRecord` — 种子标记、`nextTaskId`
- `SDLifeCategoryRecord`、`SDLifeDetailItemRecord`
- `SDWardrobeItemRecord`、`SDMenuItemRecord`
- `SDOutfitPlanRecord`、`SDOOTDHistoryRecord`、`SDOutfitPageSettingsRecord`
- `SDTodoRecord`、`SDTaskRecord`

## 7. 用户故事

1. **作为用户**，我希望在待办中添加带周重复规则的健身计划，以便每周固定跟踪。
2. **作为用户**，我希望开始/暂停待办计时并在完成时自动记录耗时，以便了解时间投入。
3. **作为用户**，我希望在「今日投入」和「本周趋势」中查看统计，以便复盘效率。
4. **作为用户**，我希望在领域里把大项目拆成子任务，以便分步推进。
5. **作为用户**，我希望在生活页管理衣物与 OOTD，以便提高衣橱利用率。
6. **作为用户**，我希望在烹饪菜单库标记招牌菜和练习中菜品，以便规划下厨。
7. **作为用户**，我希望 App 重启后数据仍在，以便长期使用同一套记录。
