# 2. 严重程度说明

| 严重程度 | 说明                                 | 示例                           |
| -------- | ------------------------------------ | ------------------------------ |
| 阻塞     | App 无法启动或核心流程完全不可用     | App 启动崩溃、无法进入首页     |
| 严重     | 核心功能异常，影响主要使用           | 待办无法保存、数据丢失         |
| 一般     | 功能可用但存在明显问题               | 页面刷新异常、部分字段显示错误 |
| 轻微     | 不影响主要流程，主要是体验或视觉问题 | 间距不统一、文字略微溢出       |

------

# 3. 优先级说明

| 优先级 | 说明           | 处理要求                         |
| ------ | -------------- | -------------------------------- |
| P0     | 发布前必须修复 | 不修复不能进入 TestFlight / 上架 |
| P1     | 重要问题       | 建议发布前修复                   |
| P2     | 一般问题       | 可排期到后续版本                 |
| P3     | 低优先级问题   | 有时间再优化                     |

# 4. Bug 状态说明

| 状态     | 说明                             |
| -------- | -------------------------------- |
| 待确认   | 已发现问题，但还未确认是否为 Bug |

| 修复中   | 正在修复                         |
