# 1. 产品概述

## 1.1 产品定位

Mindflow 是一款以个人成长为核心的生活管理 App。

当前版本已经具备分类管理、待办管理、生活模块、个人设置、底部导航、SwiftData 本地持久化等基础结构。后续产品方向是在此基础上扩展为一个综合型个人成长系统。

Mindflow 不只是普通待办 App，而是通过「待办 + 分类 + 生活领域 + 数据记录 + 复盘」的方式，帮助用户把日常行为沉淀为长期成长数据。

## 1.2 产品介绍

Mindflow 是一个帮助用户记录待办、管理分类、沉淀生活数据，并形成个人成长系统的 iOS App。

## 1.3 产品核心价值

Mindflow 的核心价值包括：

1. 帮助用户管理每日待办。
2. 帮助用户建立个人分类系统。
3. 帮助用户记录生活领域数据。
4. 帮助用户沉淀长期成长记录。
5. 帮助用户通过卡片、统计、排行和复盘看见自己的变化。

## 1.4 核心用户

Mindflow 的核心用户包括：

1. 学习型用户。
2. 个人成长用户。
3. 喜欢记录生活的人。
4. 想建立个人管理系统的人。
5. 对待办、分类、复盘、数据统计有需求的人。
6. 想管理饮食、穿搭、游戏、学习、健康等多个领域的人。

# 10. UI 与交互要求

## 10.1 整体视觉风格

Mindflow 的视觉风格应保持：

1. 清爽。
2. 柔和。
3. 卡片化。
4. 生活感。
5. 成长感。
6. 不拥挤。
7. 不过度商业化。

## 10.2 主色建议

当前底部导航和页面背景偏绿色系。

建议统一色彩：

1. 背景：白色到浅绿色渐变。
2. 主色：#6FCF97。
3. 深绿色文字：#2B5748。
4. 辅助绿色：#D8F3DC。
5. 次级文字：#6B7280。
6. 卡片背景：白色或半透明白色。

## 10.3 卡片规范

卡片应统一：

1. 大圆角。
2. 白色或浅色背景。
3. 轻阴影。
4. 标题清晰。
5. 数据突出。
6. 标签轻量。
7. 间距一致。

## 10.4 底部导航规范

底部导航要求：

1. 居中悬浮。
2. 中间添加按钮突出。
3. 当前选中 Tab 展开显示文字。
4. 未选中 Tab 只显示图标。
5. 动画自然。
6. 不遮挡页面主要操作。

## 10.5 空状态

所有主要模块需要空状态。

空状态包括：

1. 分类为空。

# 11. 非功能需求

## 11.1 性能需求

1. App 启动流畅。
2. 底部导航切换无明显卡顿。
3. 列表滚动流畅。
4. SwiftData 读写不阻塞主线程过久。
5. 大量数据下页面仍可正常使用。
6. 示例数据初始化不应造成明显等待。

## 11.2 稳定性需求

1. App 不应频繁崩溃。
2. 数据保存失败时应有日志或提示。
3. 网络请求失败时应显示错误信息。
4. 模拟数据模式下无需服务器即可运行。
5. 数据重置应有确认机制。
6. 后续正式版升级不能直接清空用户数据。

## 11.3 兼容性需求

1. 目标系统版本：iOS 17 及以上。
2. 开发环境：Xcode 15 或以上。
3. 语言：Swift 5.9 或以上。
4. 当前优先适配 iPhone。
5. 屏幕方向固定竖屏。
6. 后续可扩展 iPad 适配。

## 11.4 隐私需求

1. V1.0 不强制登录。
2. 用户数据默认保存在本地。
3. 不主动上传用户数据。
4. 如后续启用服务器或云同步，需要更新隐私政策。
5. 数据重置、导出、删除行为应由用户主动触发。

# 15. 风险与注意事项

## 15.1 当前主要风险

1. 功能模块较多，容易导致开发发散。
2. 数据模型较复杂，后续迁移风险较高。
3. 当前开发阶段存在数据全量重置逻辑，正式发布前需要谨慎处理。
4. APIClient 虽然预留服务器模式，但正式后端尚未确定。
5. 生活模块范围较大，需要明确 V1.0 边界。
6. 如果卡片过多，页面可能显得信息拥挤。

## 15.2 应对策略

1. V1.0 聚焦待办、分类、衣物、菜品、穿搭几个已具备数据结构的模块。
2. 服务器暂不作为 V1.0 必选项。
3. 正式上架前移除或限制自动清空用户数据的逻辑。
4. 每个模块先做基础闭环，再做复杂统计。
5. 优先保证数据稳定，再优化动画和视觉。
6. 每个页面都补充空状态和异常状态。

# 16. 总结

Mindflow 当前已经具备 iOS 原生 App 的基础结构，包括 SwiftUI 页面、底部导航、分类管理、待办管理、生活模块和 SwiftData 持久化能力。

V1.0 的重点应是把现有能力打磨成稳定、统一、可上架的产品版本。

当前最重要的产品原则是：

1. 先完成可用闭环。
2. 先保证本地数据稳定。
3. 先统一页面风格。
4. 先让待办、分类、衣物、菜品、穿搭模块跑通。
5. 后续再扩展饮食、游戏、目标、复盘和 AI。

Mindflow 的长期方向不是单一工具，而是一个以个人成长为核心的生活管理系统。
