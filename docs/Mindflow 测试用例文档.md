# 1. 测试目标

## 1.1 目的

验证 Mindflow 当前版本是否 **可用、稳定、数据可靠**，满足 TestFlight / 内测发布基础标准。

## 1.2 核心验证点

1. 五大 Tab 与中间「＋」按钮行为正确  
2. 待办增删改查、完成、计时、循环、统计卡片正常  
3. 分类树、穿搭、烹饪、通用详情可浏览与管理  
4. SwiftData 本地持久化：重启不丢、删除生效  
5. 空状态、大量数据、滑动交互不崩溃  
6. UI 风格统一，无明显错位遮挡  

## 1.3 当前 App 模块说明（测试必读）

| Tab 顺序 | 名称 | 对应页面 | 说明 |
|----------|------|----------|------|
| 左 1 | 待办 | `TodoView` | 主效率模块 |
| 左 2 | 生活 | `LifeView` | 当前仅「每日喝水」 |
| 中 | ＋ | — | 上下文新建（待办/分类） |
| 右 2 | 分类 | `DashboardView` | 分类树 + 穿搭/烹饪/游戏等详情 |
| 右 1 | 我的 | `ProfileSettingsView` | 设置与调试入口 |

**不适用 / 未接入主流程：**

- 「昨日任务 / 明日任务」独立列表：**无**（待办列表不按日历日切换）  
- 「领域任务 / TaskView」树形任务模块：**已移除**  
- `CategoryDetailView` + API 商品排名：**遗留代码，主 Tab 不可达**  
- 烹饪菜单 **无 UI 新增入口**（仅样本数据 + 筛选浏览）

6. 分类 → 饮食 → 烹饪 → 查看菜单库与筛选

9. 我的 → 清除所有待办  
10. 杀进程重启 → 验证数据持久化  
11. 我的 → 清除所有数据 → 重启 → 样本恢复
